                                      # 编译打包

                                      • babel - js 转译器，转换浏览器未支持的 js 特性
                                        • @babel/preset-env
                                        • @babel/preset-react
                                        • @babel/preset-typescript
                                        • core-js - js 模块化标准库，包含了大量 polyfill
                                        • bluebird - 支持 ie8 的 promise polyfill
                                        • polyfill.io - 在线自动引入 polyfill
                                        • regenerator-runtime - 用于 async 函数的独立运行时
                                      • postcss - css 的 babel
                                        • autoprefixer - 自动补全 css 前缀
                                        • cssnano - 压缩 css
                                      • webpack - v4，模块打包器，一般用于打包项目，可参考 awesome-webpack
                                        • webpack-dev-server - 快速开发
                                        • optimization - 配置优化，包括 split chunks 等
                                        • webpack-merge - 合并配置
                                        • webpack-chain - 链式生成并简化配置的修改操作
                                        • webpack-bar - 显示进度条
                                        • webpack-bundle-analyzer - bundle 分析
                                        • speed-measure-webpack-plugin - 统计 webpack 各阶段耗时
                                        • terser-webpack-plugin - 压缩 js，代替 uglify-webpack-plugin
                                        • html-webpack-plugin - 指定承载 bundle 的 html 文件
                                        • copy-webpack-plugin - 将本身存在的单个文件或整个目录复制到构建目录
                                        • compression-webpack-plugin - 压缩 asset
                                        • mini-css-extract-plugin - 提取 css 为单独文件
                                        • stylelint-webpack-plugin - 打包时检查样式
                                        • thread-loader - 放置在这个 loader 后面的 loader 会装载到 worker pool 中运行，有一定限制条件，代替 happy-pack
                                        • cache-loader - 类似于 thread-loader，该 loader 适用于性能开销较大的 loader
                                        • hard-source-webpack-plugin - 第一次打包正常速度，之后加速 90%，没有维护，建议不要再用了
                                        • raw-loader - 允许引入文件
                                        • url-loader - 解析文件引入，允许把文件转换成 base64 编码数据
                                        • file-loader - 解析文件引入
                                        • babel-loader - 使用 babel 和 webpack 编译 js 文件
                                        • ts-loader - 解析 ts
                                        • style-loader - 把 css 插入到 DOM 中
                                        • css-loader - 解析 css @import 和 url()
                                        • less-loader - 解析 less 文件成 css 文件
                                        • sass-loader - 解析 sass/scss 文件成 css 文件
                                        • postcss-loader - 使用 postcss 处理 css 文件
                                        • eslint-loader - 代码校验
                                      • rollup - 模块打包器，一般用于打包库，没接触过，可参考 awesome-rollup

                                      # 工具集

                                      • lodash
                                      • moment - 日期时间处理
                                      • dayjs - 日期时间处理，moment 的竞品
                                      • date-fns - 日期时间处理
                                      • history - 处理访问历史，比较底层的库
                                      • path-to-regexp - 将路径字符串转换成正则，比较底层的库

                                      一般用 lodash + dayjs 就能解决普遍在开发中遇到的问题，有条件的可以去学一下源码。用 dayjs 不用 moment 的原因是 moment 太大了，不用 date-fns 的原因是它没有中文文档，对国内新手不友好。
